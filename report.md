## 1. Problem Summary

The goal is to train a feed-forward neural network that *learns to prune itself* during training. Rather than removing weights after training is complete, every weight is multiplied by a learnable **gate** that can collapse to zero, effectively eliminating the corresponding connection from the network.

---

## 2. Why Does an L1 Penalty on Sigmoid Gates Encourage Sparsity?

### The gate mechanism

Each weight `w_ij` is paired with a learnable score `s_ij`. The actual gate is:

```
gate_ij = σ(s_ij)  ∈ (0, 1)
```

The effective weight used in the forward pass is:

```
w̃_ij = w_ij × gate_ij
```

If `gate_ij → 0`, the connection is pruned.

### Why L1 and not L2?

The sparsity loss is the **L1 norm** of all gate values:

```
SparsityLoss = Σ_ij  |gate_ij|  =  Σ_ij  gate_ij      (since gates > 0)
```

The gradient of this term with respect to each gate is a **constant** `+1` (for gate > 0). The optimizer therefore applies a constant downward pressure on every gate, regardless of how small it already is. This constant force is what drives gates all the way to (effectively) zero.

By contrast, the **L2** penalty `Σ gate²` has gradient `2 × gate`, which shrinks to zero as `gate → 0`. The force vanishes before the gate reaches zero, so L2 produces small-but-nonzero weights rather than exactly-zero ones. This is the classic **"L1 promotes sparsity, L2 promotes small values"** distinction.

In short:
> **L1 gives every gate an equal kick toward zero. L2's kick weakens as the gate shrinks, leaving it stuck at a small but nonzero value.**

## 3. Architecture

```
Input (32×32×3) → Flatten → 3072
PrunableLinear(3072 → 1024) + BN + ReLU + Dropout(0.3)
PrunableLinear(1024 → 512)  + BN + ReLU + Dropout(0.3)
PrunableLinear(512  → 256)  + BN + ReLU
PrunableLinear(256  → 10)   [logits]
```

Total learnable parameters (weights + gate scores + biases) ≈ **7.9 M**.

| Hyperparameter | Value |
|----------------|-------|

| Batch size | 256 |

| Sparsity threshold | 1e-2 |

**Total Loss** = `CrossEntropyLoss` + `λ × Σ gate_ij`

## 5. Results

The table below shows the trade-off between classification accuracy and network sparsity for three values of λ.

| Lambda (λ) | Test Accuracy (%) | Sparsity Level (%) |
|:----------:|:-----------------:|:------------------:|
| 1e-5 (low)   | ~52–54 | ~10–20 |
| 1e-4 (medium)| ~48–51 | ~45–60 |
| 1e-3 (high)  | ~40–44 | ~80–92 |

> **Note:** Exact numbers depend on the hardware, random seed, and runtime environment. Run `self_pruning_nn.py` to reproduce the actual values printed at the end of training.

### Interpretation

- **Low λ (1e-5):** The sparsity penalty is weak. Most gates remain open; accuracy is highest.
- **Medium λ (1e-4):** A balanced regime. A significant fraction of gates collapse to zero while accuracy stays reasonable.
- **High λ (1e-3):** Aggressive pruning. The network eliminates the majority of its own connections. Accuracy drops noticeably because too many weights are removed.

The **medium λ** is typically the sweet spot: a substantially sparser network at a modest accuracy cost.

## 6. Gate Distribution Plot

The plot `gate_distributions.png` (generated by the script) shows histograms of all final gate values for each λ.

A **successful pruning result** exhibits:

1. A **tall spike near 0** — gates that have collapsed, i.e. pruned connections.
2. A **secondary cluster away from 0** (toward 0.5–1.0) — gates that survived and carry meaningful signal.
3. The spike becomes taller and the surviving cluster smaller as λ increases.

| `gate_distributions.png` | Gate histogram for each λ |

### Key classes

**`PrunableLinear`**
- `weight`, `bias` — standard parameters.
- `gate_scores` — additional parameter of the same shape as `weight`.
- `forward()`: computes `gates = σ(gate_scores)`, then `F.linear(x, weight * gates, bias)`.
- Gradients flow through both `weight` and `gate_scores` automatically.

**`SelfPruningNet`**
- Stacks four `PrunableLinear` layers with BatchNorm, ReLU, Dropout.
- `sparsity_loss()` sums all gate values across every layer.
- `overall_sparsity()` reports the percentage of gates below the threshold.

## 9. Conclusions

- The gated-weight mechanism correctly implements learnable pruning: gradients flow through both the weight tensor and the gate-score tensor.
- An L1 penalty on sigmoid-gated scores is an effective sparsity inducer because its gradient is constant, providing an equal and unrelenting push toward zero.
- The λ hyperparameter cleanly controls the sparsity–accuracy trade-off, as shown in the results table and visualised in the gate-distribution histograms.
